# Sphinx — demo voiceover script (~2:30)

Generated to `demo/remotion/public/vo.mp3` and composited under the recording in Remotion.
Swap for a premium voice (ElevenLabs) or your own recording by replacing `vo.mp3`.

Segments (the build script generates one file per segment and concatenates, so timings are known):

1. **hook** — Remote IT support is fast. But the fix is rarely written down, and the knowledge evaporates after every ticket. Team Sphinx built an AI copilot that solves the incident and documents itself.

2. **load** — It pulls your assigned tickets straight from the Phoenix E-R-P, loads the customer's system, and connects over S-S-H — entirely under your control.

3. **diagnose** — Watch it work. The agent runs read-only diagnostics automatically — checking services, logs, and ports — and streams every step into a live trace. No black box: you see exactly what it inspects, and why.

4. **approve** — The moment it wants to change the system, it stops. Every write is gated — you approve, edit, or reject. Here it has found the root cause and proposes a minimal, lasting fix. One click, and it runs over S-S-H.

5. **validate** — It validates the result: the service is healthy, and the fix survives a reboot. The customer's problem is genuinely solved — not just patched.

6. **document** — Then it writes the documentation for you — root cause, the exact commands, and proof that it works — straight back into the E-R-P. Clean, precise, and reusable.

7. **outro** — Deterministic safety. Secrets never reach the model. Every action logged. Sphinx — the technician that never forgets to write it down.
