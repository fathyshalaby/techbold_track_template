# 04-01 Plan: Real Integration Validation

## Status

Complete

## Scope

Run precise validation probes for Phoenix, SSH, passwordless sudo, and real LLM access. This phase is evidence collection, not product implementation.

## Traceability

- Requirements: REAL-01, REAL-02, REAL-03
- Master defect map: V-002 and BL-002 for live-path proof gap, plus PLAN-02 for truthful docs separation between mock and live behavior.

## Steps

1. Inspect `.env` and `.env.example` for placeholder values without printing secrets.
2. Check Phoenix base URL reachability and credential readiness.
3. If a real Phoenix token is present, call `/api/v1/me`; otherwise record the token blocker.
4. Check SSH private key path existence and permissions.
5. If real Phoenix customer-system details and key are present, validate SSH batch connection.
6. If SSH succeeds, validate `sudo -n true`.
7. Check LLM provider credential readiness.
8. If real LLM credentials are present, run a minimal model call; otherwise record the credential blocker.
9. Store exact commands, results, and blockers in Phase 4 artifacts.

## Verification Commands

Commands are recorded in `04-VERIFICATION.md`. They do not include secret values.

## Completion Criteria

- Phoenix is validated or blocked by an exact missing token/failure mode.
- SSH and sudo are validated or blocked by exact missing key/host/failure mode.
- Real LLM access is validated or blocked by exact missing key/provider/failure mode.
- No mock-mode result is presented as live validation.
