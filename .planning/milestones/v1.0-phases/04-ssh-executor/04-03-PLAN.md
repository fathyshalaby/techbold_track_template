    ── executeApprovedCommand(command: string, target: SshTarget): Promise&lt;CommandResult&gt; ──

    1. Start timer: const startMs = Date.now().
    2. Call createSshClient(target, getEnv().SSH_PRIVATE_KEY_PATH) — may throw SSHConnectionError.
    3. Return a new Promise that wraps conn.exec with a 30s timeout:
       - Call conn.exec(`bash -lc '${command}'`, { env: { LANG: 'C' } }, (err, channel) => {…}).
       - Accumulate stdout chunks into a Buffer array; accumulate stderr chunks similarly.
       - On channel 'exit': record exitCode.
       - On channel 'close': resolve with the result (see capping below).
       - Cap: join the accumulated chunks with Buffer.concat, then slice to REDACTION_CAP_BYTES
         bytes, then call toString('utf8'). Do this for stdout and stderr independently.
       - Set a timeout with setTimeout(30_000): call channel.destroy() and resolve with
         { exitCode: null, stdout: accumulated_so_far_capped, stderr: '', durationMs, timedOut: true }.
         Clear the timeout if channel closes normally.
       - On conn.exec error: reject with the error (will surface as a run failure).
    4. After resolving/rejecting, call conn.end() in a finally-equivalent (after the promise
       resolves). Use conn.once('close', …) to ensure the connection is fully closed.

    The returned CommandResult MUST have exactly { exitCode, stdout, stderr, durationMs, timedOut }.
    No additional fields. durationMs = Date.now() - startMs (measured after channel close).
    stdout and stderr are strings (UTF-8 decoded from the capped buffer).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| executor → ssh2 channel output | stdout/stderr are untrusted bytes from a remote VM; treated as data, never as instructions |
| executor → env.ts | SSH_PRIVATE_KEY_PATH read via getEnv(); never logged; key bytes read with readFileSync and passed directly to ssh2 — not stored in any variable that reaches a log |
| caller → executor | Caller passes command string; command is already safety-gated upstream (two-gate model); executor wraps in bash -lc but does not re-validate |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-KEY | Information Disclosure | SSH private key in client.ts | mitigate | readFileSync result passed directly to ssh2 connect config; never stored in a named variable that is logged, returned, or serialised |
| T-04-INJ | Tampering | bash -lc command injection | accept | The command string arrives post-safety-gate (two-gate per SAFETY_POLICY §8); executor does not concatenate untrusted variables into the wrapper — the single template literal uses the already-gated command string only |
| T-04-A1 | Elevation of Privilege | executeApprovedCommand reachable as model tool | mitigate | grep-based test in ssh-executor.test.ts asserts the function is never referenced in ai/tools/; test must stay green after 04-05 |
| T-04-HANG | Denial of Service | Interactive prompt hang (sudo password) | mitigate | sudo -n flag enforced; never passes without -n; preflight failure is non-fatal (records sudoAvailable: false) — never blocks waiting for a TTY |
| T-04-LEAK | Information Disclosure | Raw output returned to caller | accept | Documented contract: caller applies redactSecrets before audit/UI/model; the 16 KB cap bounds exfil surface; single comment in executor.ts makes the contract explicit |
| T-04-SC | Tampering | npm/pip/cargo installs | accept | No new packages — ssh2 and @types/ssh2 already in package.json |
</threat_model>
