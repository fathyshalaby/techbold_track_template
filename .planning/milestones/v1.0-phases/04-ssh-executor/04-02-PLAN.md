  <behavior>
    All tests import from ../ssh/executor.js. The executor stub is empty (export {}) so every
    test must FAIL (RED) when this plan completes. 04-03 makes them pass (GREEN).

    Test group 1 — result shape:
    - executeApprovedCommand resolves to an object with exactly: exitCode (number), stdout (string),
      stderr (string), durationMs (number), timedOut (boolean).
    - exitCode is 0 for a successful command on the mock target.
    - durationMs is a non-negative number.

    Test group 2 — output cap:
    - When stdout from the channel exceeds REDACTION_CAP_BYTES (16384), the result.stdout length
      is at most REDACTION_CAP_BYTES.
    - When stderr exceeds REDACTION_CAP_BYTES, result.stderr length is at most REDACTION_CAP_BYTES.
    - The cap is applied per-stream independently (a large stdout does not affect stderr cap).

    Test group 3 — timeout:
    - When the command takes longer than the configured timeout, result.timedOut is true.
    - When timedOut is true, result.exitCode is null (channel was killed, no exit code received).

    Test group 4 — bash -lc wrapping:
    - The command passed to the real ssh2 exec call is wrapped as "bash -lc '<cmd>'" with LANG=C
      in the env option. Test this by inspecting the argument passed to a vi spy/mock on the
      conn.exec method. The original approvalId command is NOT passed directly to exec.

    Test group 5 — anti-pattern A1 guard (static analysis, no mock needed):
    - Use execSync('grep -r "executeApprovedCommand" ../ai/tools/ 2>/dev/null || true') in a
      vitest test. Assert the output is empty string. This proves executeApprovedCommand is never
      imported or referenced in any ai/tools/ file.

    Implementation note for writing the tests:
    Groups 1-4 require a fake ssh2 Client. Use vitest vi.mock to mock the 'ssh2' module.
    The mock Client emits 'ready' synchronously, and mock exec channels emit 'data' on stdout,
    optionally 'data' on stderr, then 'exit' with the exit code, then 'close'.
    For timeout test: the mock channel never emits 'exit' or 'close' within the timeout window —
    use vi.useFakeTimers() to fast-forward past the 30s default timeout.
    For cap test: the mock channel emits a Buffer of size REDACTION_CAP_BYTES + 1000 bytes.
  </behavior>
