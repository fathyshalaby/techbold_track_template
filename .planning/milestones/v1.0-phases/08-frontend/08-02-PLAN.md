    Export useTickets(params?: { status?: string; priority?: string; sort?: string })
    returning { tickets: Ticket[]; loading: boolean; error: string | null }.

    Implementation: useState for tickets (default []), loading (default true),
    error (default null). useEffect that calls listTickets(params) — on success sets
    tickets and clears error, on failure sets error to err.message and tickets to [].
    Always sets loading false in the finally block. The effect re-runs when params
    values change — use JSON.stringify(params) as the dependency to avoid object
    identity churn.

    On a 502 (ERP unavailable), listTickets already returns [] per api.ts — the hook
    should also set error to "ERP unavailable — showing cached empty list" so the
    component can render the banner (per ERP-05 / D-decision on empty/degraded states).
    Detect this by checking if the fetch threw AND tickets is still []: set the error
    string; do not surface raw Error objects.

    Create frontend/src/hooks/useRun.ts:

    Export useRun(runId: string | null) returning
    { run: Run | null; loading: boolean; error: string | null; refresh: () => void }.

    Implementation: useState for run (null), loading (false when runId is null, true
    otherwise), error (null), and a counter state used to trigger re-fetches.
    refresh() increments the counter. useEffect: if runId is null, do nothing;
    otherwise call getRun(runId), set run on success, set error on failure,
    set loading false in finally. Dependencies: [runId, counter].

    RunView derives pendingApproval and phase from the run object returned by this
    hook, and calls refresh() after each approve/reject/advance/abort decision and
    on relevant SSE events. The hook itself is stateless with respect to approval —
    it only fetches and caches the Run snapshot.

    Single default export is NOT used — named exports only per project conventions.

    Create frontend/src/hooks/useRunEvents.ts:

    Export useRunEvents(runId: string | null) returning
    { events: SseEvent[]; connected: boolean }.

    This hook provides the live event stream only. Components that also need
    pendingApproval and phase must call useRun(runId) separately and call its
    refresh() on relevant SSE events (e.g. approval.required, command.executing,
    run.completed, run.failed). useRunEvents does NOT expose those fields.

    Implementation:
    - useState for events (default []) and connected (default false).
    - useEffect: if runId is null, return immediately.
      Import getEventsUrl from "../api" and construct the URL as getEventsUrl(runId).
      Do NOT inline VITE_API_BASE — always use the exported helper.
      Create new EventSource(url).
    - On EventSource open: setConnected(true).
    - On EventSource error: setConnected(false). Do NOT close the EventSource
      on error — the browser will retry automatically.
    - Register one addEventListener for EACH of the 14 SSE_EVENT_TYPES (not a
      generic onmessage). The "keepalive" event from the server is NOT in
      SSE_EVENT_TYPES — do not register a listener for it, so it is silently
      ignored. For each registered event type, the handler parses event.data as
      JSON and appends a SseEvent to events via functional setState:
      setEvents(prev => [...prev, parsed]).
      The parsed shape must match SseEvent: { type, runId, ts, payload }.
    - Cleanup: return a function that calls es.close() and setConnected(false).
    - Dependency array: [runId].

    The hook must NOT start an EventSource when runId is null — this prevents a
    connection attempt before a run exists.

    No default export.
  </action>
