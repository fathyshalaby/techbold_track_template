owns all run-phase transitions without touching I/O. This is the state machine skeleton; the async
driver (Plan 05) performs effects.

Every transition is a table-driven assertion. Correctness here means correctness everywhere.

Output: Exported reduce() function, OrchestratorState/OrchestratorEvent/SideEffect types, MAX_STEPS
constant, and a complete test suite covering all transitions including the max-steps cap.

      (Import DiagnosticProposal, FixProposal, ValidationResult from agent files via 05-01 types file)

    SideEffect — discriminated union:
      { type: 'createPendingApproval'; runId: string; proposal: DiagnosticProposal | FixProposal }
      { type: 'appendAuditEvent'; runId: string; auditType: string; actor: AuditEvent['actor']; payload: unknown }
      { type: 'appendObservation'; runId: string; source: Observation['source']; content: string }
      { type: 'emitEvent'; runId: string; eventType: string; payload: unknown }
      { type: 'updateRunPhase'; runId: string; phase: RunPhaseValue }
      { type: 'updateRunStatus'; runId: string; status: RunStatusValue }

    ReducerResult: { nextState: OrchestratorState; sideEffects: SideEffect[] }

    MAX_STEPS = 12 (exported const)

    reduce(state, event) logic:
    - Terminal guard first: if phase is COMPLETED/FAILED/ABORTED → return { nextState: state, sideEffects: [] }
    - Max-steps guard: if event.type is diagnostic_proposal_ready or fix_proposal_ready AND
      state.stepCount >= MAX_STEPS → transition to WAITING_FOR_ACTIVITY_REVIEW, emit run.steps_capped audit event
    - Switch on state.phase + event.type combinations per the transition table above
    - Every transition that advances stepCount: increment on diagnostic_proposal_ready and fix_proposal_ready events
    - Helper buildState(state, overrides): returns new OrchestratorState (no mutation)
    - Helper buildAuditEffect(state, auditType, actor, payload): returns SideEffect of type appendAuditEvent
    - Helper buildPhaseEffect(state, phase): returns SideEffect of type updateRunPhase

    DO NOT import from store/audit.ts, store/runs.ts, ssh/executor.ts, or events/run-event-bus.ts.
    The reducer is pure — it only imports types and schema enum values.
