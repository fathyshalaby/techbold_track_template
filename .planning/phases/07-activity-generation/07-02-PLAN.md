| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-07-04 | Information Disclosure | `/submit` body fields (technician edits) â†’ Phoenix | mitigate | Each override field passed through `redactSecrets` before inclusion in `ActivityCreate` body â€” same defence-in-depth as draft fields |
| T-07-05 | Tampering | Duplicate Phoenix activity â€” non-idempotent POST retried | mitigate | `createActivity` is never wrapped in the retry helper (existing client policy comment preserved); route returns 502 on failure without retry â€” duplicate ERP record risk eliminated |
| T-07-06 | Elevation of Privilege | Phase guard bypass â€” draft called from CREATED/TRIAGING | mitigate | Explicit allowlist of phases (WAITING_FOR_ACTIVITY_REVIEW, DRAFTING_ACTIVITY, COMPLETED); all others return 409 before agent call |
