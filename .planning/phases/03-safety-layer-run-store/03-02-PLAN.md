  <action>
    Export `REDACTION_CAP_BYTES = 16384` as a named const.

    Export `RedactionPattern` type: `{ name: string; pattern: RegExp; replacement: string | ((match: string, ...groups: string[]) => string) }`.

    Define an internal `REDACTION_PATTERNS` array of `RedactionPattern`. Implement every §6 category as a labeled pattern. Pattern names (used for internal tracing, not exposed to callers):
    - `private-key-block` — matches `-----BEGIN ... PRIVATE KEY-----` … `-----END ... PRIVATE KEY-----` (multiline, dotall). Replacement: `«redacted»`.
    - `password-field` — matches `password\s*=\s*\S+` and `passwd\s*=\s*\S+`, case-insensitive. Replacement: preserve key name + `=«redacted»`.
    - `token-field` — matches `token\s*=\s*\S+`, case-insensitive. Replacement: preserve key + `=«redacted»`.
    - `secret-field` — matches `secret\s*=\s*\S+`, case-insensitive. Replacement: preserve key + `=«redacted»`.
    - `api-key-field` — matches `api[_-]?key\s*=\s*\S+`, case-insensitive. Replacement: preserve key + `=«redacted»`.
    - `authorization-header` — matches `Authorization\s*:\s*.+` (to end of line), case-insensitive. Replacement: preserve `Authorization: ` prefix + `«redacted»`.
    - `bearer-token` — matches `Bearer\s+[A-Za-z0-9\-_.~+/]+=*`, case-insensitive. Replacement: `Bearer «redacted»`.
    - `db-connection-string` — matches `(postgres(?:ql)?|mysql|mongodb|redis):\/\/[^@\s]+@[^\s]+`. Replacement: preserve scheme + `://«redacted»`.
    - `aws-access-key` — matches `AKIA[A-Z0-9]{16}`. Replacement: `«redacted»`.
    - `env-secret-var` — matches `\b(?:[A-Z_]*(?:SECRET|TOKEN|KEY|PASS|PASSWORD|CREDENTIAL)[A-Z_0-9]*)\s*=\s*\S+`. Replacement: preserve key name + `=«redacted»`. Apply this pattern AFTER the more specific ones to avoid double-redacting.
    - `azure-sas-fragment` — matches `sig=[A-Za-z0-9%+/=]{20,}`. Replacement: `sig=«redacted»`.

    Export `redactSecrets(text: string): string`. Implementation:
    1. Cap: if `text.length > REDACTION_CAP_BYTES`, slice to `REDACTION_CAP_BYTES` before any processing.
    2. Apply each pattern in `REDACTION_PATTERNS` in order using `String.prototype.replace` with the `g` flag (and `s` flag for multiline key blocks). Since patterns are applied sequentially on the result of the previous, the function is compositional.
    3. Return the final string.

    The function is pure: no imports from routes/, ai/, or store/. No I/O. No module-level state beyond the frozen pattern array. ESM `.js` suffix on any future imports. No fenced code in action.

    Commit: `feat(03-02): implement secret redaction pure function`.
  </action>
