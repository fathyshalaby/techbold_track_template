    Replace frontend/src/App.tsx with the view-state shell:

    State:
    - currentView: 'list' | 'run' | 'activity' — useState default 'list'
    - activeRunId: string | null — useState default null
    - activeTicketTitle: string — useState default ''
    - createError: string | null — useState default null
    - creating: boolean — useState default false

    The shell renders:
    - When currentView === 'list': TicketListView with an onSelectTicket callback.
    - When currentView === 'run' and activeRunId is non-null: RunView with runId,
      ticketTitle, and onActivityReady callback that sets currentView to 'activity'.
    - When currentView === 'activity' and activeRunId is non-null: ActivityView with
      runId and onDone callback that sets currentView back to 'list' and clears
      activeRunId.

    onSelectTicket(ticket: Ticket):
    - Sets creating to true, clears createError.
    - Calls createRun(ticket.id) from api.ts.
    - On success: sets activeRunId to response.runId, sets activeTicketTitle to
      ticket.title, sets currentView to 'run'.
    - On failure: sets createError to the error message string.
    - Always sets creating to false in finally.

    If createError is set, render it as a small error banner above the current view
    (class "error-banner") that the user can dismiss (button sets createError null).

    A thin header bar renders "AI Service Desk Autopilot" as an h1 and, when
    currentView !== 'list', a "← Back to tickets" button that sets currentView to
    'list' and clears activeRunId.

    No router library. No CSS framework. Use class names for styling hooks; the CSS
    is filled in by Task 2.
