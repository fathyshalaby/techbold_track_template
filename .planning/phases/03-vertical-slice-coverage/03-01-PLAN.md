# 03-01 Plan: Vertical-Slice Coverage

## Status

Planned

## Scope

Create deterministic backend coverage for the critical service desk workflow after the Phase 2 browser UAT found and fixed the built-in mock agent path.

## Traceability

- Requirement: E2E-01
- Requirement: PLAN-01
- Master defect map: V-002, BL-002

## Steps

1. Add a focused vertical-slice test under `backend/src/tests`.
2. Exercise the real app routes instead of isolated handler spies.
3. Consume the SSE route output and assert a lifecycle event reaches the stream.
4. Approve a diagnostic command with an edit and assert the command result uses the edited command.
5. Continue through fix approval, validation, activity draft, activity submit, and final run status.
6. Run focused coverage and backend typecheck.
7. Record verification evidence and update roadmap, requirements, and state.

## Production Change Policy

No production source changes are planned. If the test exposes a defect, any production change must cite `.planning/audits/V1.1-MASTER-DEFECT-MAP.md` and stay limited to the failing workflow.

## Verification Commands

```bash
pnpm --dir backend test -- vertical-slice
pnpm --dir backend typecheck
```

## Completion Criteria

- The vertical-slice test fails for real workflow regressions, not merely source text or mock wiring.
- The test covers run creation, SSE event consumption, edited approval execution, validation, and activity submission.
- No untraced production change is introduced.
