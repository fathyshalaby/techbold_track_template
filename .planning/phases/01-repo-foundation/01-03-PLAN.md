<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| docker build context → image | Dockerfile copies only source; no .env or keys/ in the image |
| HTTP client → Hono routes | All route input untrusted; health route has no user input so attack surface is zero |
| CORS * (open) | Intentional for single-machine local tool; documented accept |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-03-01 | Information Disclosure | Dockerfile COPY context | mitigate | .dockerignore (created in Plan 01) excludes .env, keys/, node_modules/. Verify with docker build --no-cache and inspect image layers — no secret files present. |
| T-03-02 | Information Disclosure | app.onError response | mitigate | onError returns err.message only, not err.stack. In production the stack could leak internal paths; for this local tool err.message is acceptable. Do not return raw Error objects. |
| T-03-03 | Spoofing | CORS open (*) | accept | Single-machine local tool, no auth cookies, no sensitive same-origin resources. Documented per ARCHITECTURE.md §10. Not a finding — a conscious decision. |
| T-03-04 | Denial of Service | /health read of isMockMode | accept | Pure synchronous read of already-parsed env object; no I/O, no user input. No DoS vector. |
| T-03-SC | Tampering | npm/pip/cargo installs | accept | No new installs in this plan. All deps declared in Plan 01. |
</threat_model>
