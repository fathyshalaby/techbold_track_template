    - On 5xx or network error (fetch throws): retry once after 200ms sleep; if second attempt also fails, throw PhoenixNetworkError
    - On 4xx other than 401/404/422: throw PhoenixNetworkError with status code in message

    Error classes: PhoenixAuthError, PhoenixNotFoundError, PhoenixValidationError, PhoenixNetworkError — all extend Error with name set to class name.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Backend → Phoenix ERP | Outbound HTTP; bearer token must stay server-side |
| Phoenix response → typed domain | Untrusted JSON from external API; must be Zod-parsed |
| Caller-supplied ticketId → URL path | Numeric ID must be validated before URL construction |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-03 | Information Disclosure | PhoenixClient — PHOENIX_API_TOKEN | mitigate | Token only in Authorization header; never logged, never in error messages, never returned to caller; client constructed with explicit values not raw env strings exposed to caller |
| T-02-04 | Tampering | PhoenixClient — response parsing | mitigate | All responses parsed through Zod schema; .parse() throws on mismatch — never use `as T` cast |
| T-02-05 | Elevation of Privilege | ticketId path parameter | mitigate | Validate ticketId is integer > 0 before URL construction; throw TypeError if not — prevents path traversal (e.g. ticketId='../me') |
| T-02-06 | Denial of Service | fetch with no timeout | mitigate | AbortController with 8000ms applied to every request; abort signal passed to fetch options |
| T-02-07 | Spoofing | Retry on 4xx | mitigate | Retry only on 5xx/network; 401/404/422 never retried — prevents hammering auth endpoint |
| T-02-SC | Tampering | npm installs | accept | No new packages added in this plan; existing package-lock.json unchanged |
</threat_model>
