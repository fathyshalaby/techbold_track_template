    ticketsRouter: new Hono(). Three routes:

    GET / (list):
    - Parse query params with Zod: status (optional TicketStatusSchema), priority (optional string),
      sort (optional enum 'date'|'priority'|'status'). Use @hono/zod-validator or manual c.req.query().
    - Construct client: resolveClientMode('phoenix') === 'mock' ? new MockPhoenixClient() : new PhoenixClient(getEnv().PHOENIX_API_URL, getEnv().PHOENIX_API_TOKEN).
    - Call client.listTickets({ status, priority, sort }).
    - On success: return c.json(tickets, 200).
    - On PhoenixAuthError: return c.json({ error: 'upstream authentication failed' }, 502).
    - On PhoenixNotFoundError: return c.json([], 200). (empty is valid for list)
    - On PhoenixNetworkError: return c.json({ error: 'ERP unavailable' }, 502).
    - On any other error: let it bubble to app.onError (→ 500 JSON).

    GET /:id (single ticket):
    - Parse id param: parseInt(c.req.param('id'), 10). If NaN return c.json({ error: 'invalid ticket id' }, 400).
    - Call client.getTicket(id).
    - On success: c.json(ticket, 200).
    - On PhoenixNotFoundError: c.json({ error: 'ticket not found' }, 404).
    - On PhoenixAuthError: c.json({ error: 'upstream authentication failed' }, 502).
    - On PhoenixNetworkError: c.json({ error: 'ERP unavailable' }, 502).

    GET /:id/customer-system:
    - Same id parsing as above; 400 on NaN.
    - Call client.getCustomerSystem(id).
    - On success: c.json(customerSystem, 200).
    - On PhoenixNotFoundError: c.json({ error: 'customer system not found' }, 404).
    - On PhoenixAuthError: c.json({ error: 'upstream authentication failed' }, 502).
    - On PhoenixNetworkError: c.json({ error: 'ERP unavailable' }, 502).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser/client → GET /api/tickets | Untrusted query params (status, priority, sort, id) |
| Route handler → Phoenix ERP / mock | Upstream calls; bearer token must not appear in responses |
| Phoenix response → route response | Typed Zod response from Plan 01; safe to pass through |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-08 | Information Disclosure | GET /api/tickets — error responses | mitigate | Error JSON returns only a generic message string; PhoenixAuthError detail and token value are never included in response body |
| T-02-09 | Tampering | ticket_id path param | mitigate | parseInt + NaN check before any use; non-integer returns 400 before the client is called; mirrors client.ts validation |
| T-02-10 | Information Disclosure | PHOENIX_API_TOKEN in route module | mitigate | getEnv() called lazily inside handler, not at module level; token only passed to PhoenixClient constructor, never serialised or logged |
| T-02-11 | Denial of Service | Unbounded query string values | accept | status/priority/sort are passed to Phoenix which applies its own server-side validation; 422 from Phoenix surfaces as 502 to caller |
| T-02-12 | Spoofing | MockPhoenixClient in production | mitigate | Client selection uses resolveClientMode('phoenix') which requires MOCK_MODE or MOCK_PHOENIX to be true; false by default in production env |
| T-02-SC | Tampering | npm installs | accept | No new packages added; package-lock.json unchanged |
</threat_model>
