| Boundary | Description |
|----------|-------------|
| AI agent → proposeSshCommand | Agent provides command string as untrusted structured input; tool has no execute — proposal is recorded, not run |
| backend → executeApprovedCommand | Called only by deterministic backend code after human approval; never reachable via model tool dispatch |
| createSshExecutor factory | Reads resolveClientMode from env; no user-controlled input |

| T-04W-A1 | Elevation of Privilege | executeApprovedCommand registered as model tool | mitigate | Task 2 guard test asserts it; test is permanent CI gate — red on violation |
| T-04W-INJ | Tampering | Prompt injection via proposeSshCommand command field | accept | proposeSshCommand has no execute; the command string reaches safety gate + human approval before any execution, never reaches ssh2 directly from the tool |
| T-04W-ENV | Information Disclosure | resolveClientMode reads MOCK_SSH from env | accept | env.ts already parses and validates all env vars; no secret transmitted |
| T-04W-SC | Tampering | npm/pip/cargo installs | accept | No new packages added in this plan |
